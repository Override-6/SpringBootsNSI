Languages de Haut niveau

Les langages de haut niveau sont des languages qui "s'éloignent" de la machine pour ce raprocher de l'utilisateur; Un language de haut niveau, permet à l'utilisateur d'effectuer des actions complexes en se souciant de peux de choses, comme le type de processeur, ou encore le système d'exploitation.

Les languages de haut niveau, pendant qu'ils sont éxécutés (Le runtime), n'existent pas pour la plus part (mise à part, par exemple le python, qui est directement interprété).

Pour certains languages, ils sont traduits par un compilateur, qui va se charger de transcrire le code de haut niveau en un code de plus bas niveau. Ce code peut être directement lu par l'os (comme le ByteCode), ou par un interpreteur, qui va se charger d'interpréter le code traduit, pour ainsi effectuer des appels au système d'exploitation

Prenons l'exemple du java, qui est un language de haut niveau.

Le java, écrit dans un fichier au format ".java", sera détruit dés lors que ce fichier sera compilé. Un compilateur va se charger de traduire le code source ".java" en ByteCode, qui sera stoqué dans un ficher ".class".

De là, ce fichier ".class", pour être éxécuté, a besoins d'une machine virtuelle, pour le cas d'un ByteCode provenant de javac, il lui faudra une JVM (Java Virtual Machine). Chaque architecture d'ordinateur a une version spécifique de la JVM, écrite en C/C++ (Exemple: HotSpot)

Une fois la JVM démarrée, elle interprètera les instructions du bytecode, et traduira le résultat obtenu lors de l'execution en language machine (en binnaire)
